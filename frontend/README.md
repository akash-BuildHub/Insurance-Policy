# PolicyMate AI — Frontend (React + Vite SPA)

A pure single-page React app that talks to the FastAPI backend over REST.

## Stack

- **React 19** + **Vite 7** (no SSR)
- **TanStack Router** (file-based routing) + **TanStack Query** (server cache)
- **Tailwind CSS v4** + **shadcn/ui** + **lucide-react**
- **Zod** for form validation
- **jsPDF** + **jspdf-autotable** for the downloadable recommendation report
- **Recharts** for admin charts

## Setup

### Prerequisites

- Node.js 20+ and npm

### Install

```bash
cd frontend
npm install
```

### Configure

Copy `.env.example` to `.env` (optional — the default points to `http://localhost:8000`):

```env
VITE_API_URL=http://localhost:8000
```

### Run

```bash
npm run dev
```

This single command starts **both** services side-by-side via `concurrently`:

| Process  | URL                          | Command                                                     |
| -------- | ---------------------------- | ----------------------------------------------------------- |
| backend  | <http://localhost:8000>      | `node scripts/dev-backend.mjs` → uvicorn (auto-detects venv)|
| frontend | <http://localhost:5173>      | `vite`                                                      |

`scripts/dev-backend.mjs` picks the Python interpreter in this order:

1. `$POLICYMATE_PYTHON` env override
2. `../backend/.venv/Scripts/python.exe` (Windows venv)
3. `../backend/.venv/bin/python` (Unix venv)
4. system `python` on PATH

`Ctrl+C` once stops both processes cleanly. If you'd rather run them in separate
terminals:

```bash
npm run dev:backend     # backend only
npm run dev:frontend    # frontend only (= plain `vite`)
```

### Build

```bash
npm run build
npm run preview
```

## Auth model

- JWT issued by the backend is stored in `localStorage` under `policymate.token`.
- Every authenticated request goes through `src/lib/api.ts` which attaches
  `Authorization: Bearer <token>`.
- `<AuthProvider>` (in `src/lib/auth.tsx`) on mount calls `GET /api/auth/me` to validate
  the token and load the current user.
- Routes nested under `_authenticated` show "Loading…" briefly while the token is being
  validated, then redirect to `/login?redirect=…` if there's no user.

## Admin link

The header shows an **Admin** link when `user.role === "admin"`. The first sign-up that
matches `INITIAL_ADMIN_EMAIL` (configured in the backend `.env`) becomes the admin
automatically. Alternative: run `UPDATE users SET role='admin' WHERE email='…';`.
